## 1 对象与类

### C++的语言特色
增加的重要机制：
* 设计方法：函数模板、类模板
* 数据保护机制：const，封装
* 代码重用：运算符/函数重载，继承，派生
重要思想：**数据抽象**
**数据类型与操作分离**，不同数据类型适用同一个函数

![](https://cdn.luogu.com.cn/upload/image_hosting/ve6vig85.png)
新的字符串类型string：区分读入方式get和getline

### 内联函数机制inline
编译时对简单函数不再执行调用函数机制，直接替换程序段，加快了效率

但不能太复杂，比如不能递归

### 内存、指针、引用
`new`分配内存（=malloc），`delete`释放内存（=free），也可对数组等操作

指针：指向变量地址（比如new变量的地址），可以更改指向的地址

数组的本质是指针，数组变量指向首个元素的地址，可以用指针直接访问各个元素

地址传递&：除了传值传参，还能传递变量地址，可以直接改变原变量数值

引用&：相当于给变量取别名，直接改变变量值，不能再更改指向（指针能改）

### 数值量
`const`：常量，不能再更改值；传参加const也能保护值不更改
![](https://cdn.luogu.com.cn/upload/image_hosting/2m4wl717.png)
作用域：优先低级，局部掩蔽全局，可以用`::`指定域区分量

### 面向过程与面向对象

**面向过程**：

* 程序=算法+数据结构
* 程序是子功能的组合，设计*自顶向下*划分，以*函数*为中心
* 函数与数据分离，数据公用，函数和数据互相存取调用
问题：
> 修改程序复杂，不适合大型项目，重用性差；数据安全保障弱；调试测试复杂

**面向对象**

* 对象：“具象的事物”，和实际物品、内容对应
* 类：对象的实现方式为*类的封装*。类包含*数据*和*函数*，用函数进行操作
* 设计程序步骤：类的设计和封装、函数调用

* 程序=对象+消息
* 程序是对象组成的系统，以*对象*为中心

> 属性相似的对象之间可以用派生实现，提高代码重用性

### 类的定义

**对象**是构成一个系统的基本单位：具备**属性**和**行为**要素，由**数据**（属性）和**函数**（方法）构成；

类是对象的描述：类是数据和函数的**封装**

**类是对象的抽象，对象是类的实例**

类的特征：**封装、继承、多态**

binding束定：静态束定在编译时完成（如变量），动态束定在运行时才完成（如虚函数）

### 类的声明
先声明类，再实例化（定义）对象；定义的对象，数据分别存储，函数共同存储
```cpp
class xxx
{
    public:

    private:

};
```
作用域：`public`公有，外界可以访问；`private`私有，仅类内函数可访问，外界不可访问；`protected`保护，仅本类和派生类内函数可访问

类内可包含数据变量和成员函数；引用类内成员，用类名.成员名或指针->成员名

## 2 类的成员，相关的函数与操作
### 成员函数
成员函数在类内声明，可以在类内外定义

离开类（定义时），需要用类名::指定
#### 重载
同一函数名用不同类型的参数，一般实现相同功能

不能有歧义：必须用不同参数类型、个数区分，不允许相同参数不同返回值类型，不允许参数缺省后出现相同参数函数
#### 特殊的成员函数
**构造函数**：创建对象时调用，用于初始化，分配内存

**析构函数**：对象使用结束（delete）时调用，用于释放内存

都有默认的空函数，没有返回值和函数名，直接用类名，~类名指代，不能主动调用

可以重载构造函数，实现不同参数的不同初始化，注意定义构造函数后就没有默认空函数，不能无参数构造

**复制构造函数**：只用于两种语句：类 b(a)或类 b=a

参数为一个同类型类的引用，用于用已有的对象复制对象
只用于新对象定义的复制过程，只赋值复制不用复制构造函数，直接复制

复制构造函数的3种调用场景：用一个对象初始化另一个新定义的对象，调用函数传递形参对象，函数结束返回值对象

> 如果传参的是引用，就不需要复制构造，直接共享地址

*调用顺序*问题：
* 先构造的后析构
* 局部对象离开作用域就析构
* 函数返回类时，若未同时创建新对象，需要复制构造临时对象返回上级，函数结束（内部类都析构）后再析构
#### 浅复制问题
出现在**指针复制**过程，数组复制为深复制

直接复制，新变量获得旧变量地址，仍然指向同个变量，不独立，同时修改值，导致混乱

解决方案（深复制）：定义复制构造函数，新申请变量地址，把原地址变量的值传给新地址变量
### 对象数组
注意构造时要对每一个调用构造函数传参

**对象指针** 类似普通指针

`this`：对象内部代指自己的指针，用于访问自己的成员

**向前引用**

### 数据，保护机制
#### 常量机制
前面加const

常成员：成员前加const

常指针：int* const xxx，const在数据前表示指针指向的数据类型是常量，在指针前表示指针本身是常量

常引用：转变为常对象，避免数据修改，变量参数加在前

*常成员函数*：函数的标记，不允许修改数据成员，加在函数声明后

**常对象只能调用常成员函数**（用到的都得不变，保证安全性）

`mutable`：在常成员函数修改数值的变量标记

![](https://cdn.luogu.com.cn/upload/image_hosting/cc0dmnlg.png)
#### 静态机制
`static`标记，对类的不同对象共用，是不同对象交流的接口

地址共用，声明类时就分配，不需等到定义对象，作用到程序结束

只能在主程序外初始化，不能用构造函数，不能用`this`（不属于对象）

静态成员函数只能访问静态成员

#### 友元

标记在类声明内进行

友元关系是单向非传递的
#### 嵌套类
类属于另一个类，所有操作只能在上级类完成
#### 组合类
类之间是同级的，不过一个类使用了另一个类，实例化该类的对象

构造顺序：由内向外
#### 链表类

### 运算符重载
已有的运算符赋予多重含义，对类（数据类型）加入新功能；优先级和性质不变；类似函数重载，传参构造机制相同

加入关键字`operator`，当作正常函数定义写：`bool operator <(classa &x,classa,&y)`

可选择重载为成员函数（在类声明内重载）或友元函数（加friend）

成员重载省略第一个参数，默认为同类型对象，用this指代

指定重载方式的运算符 成员：赋值，下标，函数调用；友元：流，类型转换；不能重载：如指针

一般加const &常引用保护参数

（特殊）单目后置运算符重载：指定一个int类型但无名的形参表示后置，不适用

一般单目成员重载，双目友元重载

*=重载*：注意深复制

*流重载*：<<，>>，用于规范类的输出和输入，搭配cin，cout

参数和返回值必须是I/O对象的引用：`istream/ostream &input/output`

*返回左值*：返回类型为引用时，有一个可改变的对象（地址）对应，可以直接被赋值

*类型转换运算符*：默认低精度转高精度，否则精度损失
可以利用构造函数作为类型转换

**类型转换函数** 格式：`operator type()`，没有参数与返回类型，只能作为成员

**参数传递问题**：执行从左到右，传参从右到左，栈
单目后置运算符：不能返回引用，会导致对象还没利用就被析构

### 继承、派生、组合
在用已有的类构造新的类的过程中，

* 继承：保持原类特性，实现代码重用

* 派生：新增特性

*继承方式*

所有的继承方式，基类的私有成员都不可见

* `public`：公有和保护属性不变
* `private`：公有和保护都变成私有成员
* `protected`：公有和保护都变成保护成员，也就是外界不可见
![](https://cdn.luogu.com.cn/upload/image_hosting/3obsebl3.png)
![](https://cdn.luogu.com.cn/upload/image_hosting/q2d8e3sz.png)

有必要访问基类私有成员时，可把派生类设为基类友元

可以把基类成员的访问属性直接调低，或用`using`调整非私有成员

派生类声明方法：class 派生类:继承方式1 基类1,继承方式2 基类2…

获得基类的成员（遗传），同时可以定义新的成员（变异）

*重名问题*：优先使用派生类成员，可以用::指定基类成员

*构造与析构*：

不继承构造和析构函数，但要先从内向外，从前往后构造基类成员，再构造派生类成员

因此可以在派生类构造函数调用基类构造函数，在声明头用:基类名1（参数1）,基类名2 参数2…表示调用

复制构造相同，析构相反

**多继承**：同时从多个基类派生

多重派生时，只有直接派生关系成立，需要逐层向内调用内层基类成员

多类派生时，按顺序处理基类

*二义性问题*：类名::限定

多层多类结合派生的二义性比较麻烦，因为可能有通过不同中间基类继承的两个相同根基类对象

解决方法：**虚基类机制**

关键字`virtual`，加在继承方式前，遇到此类相同根基类，只继承一次

需要用在该虚基类的所有直接派生类，以避免重复继承；初始化由最后派生类直接完成

*构造函数执行顺序*：长辈优先，虚基类优先，顺序优先

*类型兼容*：派生类可以转为基类，或被基类指针、引用指定

**组合**：一个类作为另一个类的成员，子对象关系
也需要一同构造，按序构造
### 多态
含义：同样的消息被不同对象接受表现不同的行为（联系**动态束定**）一个接口，多种功能

包含：重载多态，强制多态（类型转换），包含多态（虚函数），参数多态（模板）

* 静态多态：编译时确定，函数重载
* 动态多态：运行时确定，虚函数
#### 虚函数
关键字`virtual`，在成员函数声明前

基类和派生类的同名函数，都加virtual

有时候基类不定义函数，只确定参数和返回值，声明时后面加=0，无函数体，为**纯虚函数**

包含纯虚函数的类为抽象类，无法直接定义对象；只有所有纯虚函数都定义了相应的函数体的类才能实例化

**所有**派生类各自定义同名函数，参数和返回值必须相同，必须在每个派生类都定义，不然出现无法束定的错误

不能成为虚函数：非成员，静态函数，inline，构造函数

调用时就可以根据类的不同选择该函数的不同功能

**虚函数表**：指针数组，包括同一组各个虚函数的入口地址，运行时选择具体的函数

## 3 流、模板与其它工具
### 流
“从一个对象到另一个对象的数据流动的字节序列”

输入：输入设备流向内存缓冲区

输出：内存缓冲区流向输出设备

流对象与输入输出设备连接，通过操作流对象完成输入输出

流类包括：通用，文件，字符串的输入，输出，输入输出类

输出流对象：cout,cerr错误输出（无缓冲），clog错误输出（有缓冲）

`iomanip`输出格式控制：进制，`fixed`定点数，`setprecision`精度，`setfill`填充字符 等

输入流对象cin

get与put，getline

文件流对象ifstream,ofstream；二进制文件binary

字符串流istrstream,ostrstream
### 模板template
用于实现多态，相同程序处理不同类型的对象，可用于函数、类等

`template <typename T>`或`template <class T>`，写在声明之前，下面声明时可以使用类型T

函数模板必须为全局作用域，类型必须匹配

调用时优先匹配已经存在的函数，再尝试通过类型转换匹配存在的函数，最后匹配模板

需要避免可能的歧义和匹配失败

模板实参：调用时用<>手动指定模板的类型

*常量参数化*：定义模板时定义一些参数量，调用的时候用<>指定值

类模板的参数可以设为类class或数据类型typename

类模板是类的抽象，类是类模板的实例

数组类模板
### 标准模板库STL
**命名空间**：用::指定所属，可用`namespace`将一些量划入一个命名空间，用`using`使用

> 关于using namespace std;：标准C++库的功能所在的命名空间std，使用时需要调用

STL属于泛型程序设计（相同程序在不同类型间通用）

包含：**容器，迭代器，函数对象，算法**

容器：如set,map,vector等

* 顺序容器，关联容器（无顺序，用于快速组织数据）

* 迭代器：访问容器内元素的工具，如iterator,begin(),end()，本质是指针

> 可分为输入迭代器/输出迭代器，前向迭代器/双向迭代器/随机访问迭代器等

* 适配器：接口类，包含容器适配器(如栈和队列)，迭代器适配器和函数对象适配器
* 算法：基于模板实现的方法sort就是算法库中的排序算法
> less<int>(),greater<int>()：重载的比较函数；也可写自定义函数

### 异常
程序无法继续执行的情况为异常，可以用异常控制介入

好处是增强健壮性，减少可能的错误，**发现与处理异常分开机制**

异常检查`try`，异常定义与抛出`throw`，异常捕捉与处理`catch`

断言`assert`：不满足某逻辑条件时报错终止

`debug`与`release`：分别用于调试和正式发行，一些功能（如断言）只在debug使用
